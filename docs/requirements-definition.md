# 要件定義書
# Ver 1.1.0

:::note info
要件定義に限定した第2稿（基本設計要素を分離）
:::

## 1. システムの目的
サークル GeeKEN の共有資産である技術書の貸出・返却を管理し、貸借状況の可視化と返却遅延の通知を実現する。

## 2. 対象範囲
本ドキュメントの対象範囲は以下とする。

- 書籍の貸出
- 書籍の返却
- 貸出状況の確認
- 書籍コメント
- Discord 通知（返却リマインド、期限超過通知）

## 3. ステークホルダー

### 3.1 一般会員
- ログイン
- 本一覧・貸出状況の確認
- 貸出・返却
- コメント投稿
- 通知設定（リマインド送信時刻）

### 3.2 管理者
- 書籍情報の管理
- 貸出状況の監視
- 期限超過の確認

### 3.3 システム
- 貸出可否判定
- 貸出期限判定
- Discord への通知送信

## 4. 技術前提
- フロントエンド: React / TypeScript / Tailwind CSS
- バックエンド: NestJS
- データベース: PostgreSQL
- 認証: GitHub OAuth（GeeKEN GitHub Organization メンバー判定を含む）
- 外部連携: DevelopersAPI, Discord

## 5. 機能要件

### 5.1 ユーザー認証・利用条件
- 利用者は GitHub OAuth によって認証されること
- 利用者は GeeKEN GitHub Organization のメンバーであること
- 利用者は `User.isActive = true` であること
- 利用者は初回ログイン時に GitHub 認証とあわせて Discord 連携を完了し、`discordId` が登録済みであること

### 5.2 貸出要件
- 同時貸出上限は 1 人 1 冊とする
- 貸出期限は貸出確定日時から 7 日後とする
- 貸出延長は原則として提供しない
- 未返却または期限超過の貸出があるユーザーは新規貸出不可とする
- 貸出対象外の書籍は貸出不可とする
- 既に他ユーザーへ貸出中の書籍は貸出不可とする

### 5.3 返却要件
- 返却完了により当該貸出は終了状態となる
- 返却完了後の追加ペナルティは課さない

### 5.4 通知要件
- 通知チャネルはサークル Discord チャンネルとする
- 通知は Bot 経由で送信する
- メンション対象は当該書籍の借用ユーザー本人のみとする
- 通知タイムゾーンは JST 固定とする
- リマインド通知は期限前に 1 回のみ送信する
- リマインド通知時刻はユーザーがアプリ上で設定できるものとする
- リマインド通知時刻の設定可能範囲は「返却期限日の前日 18:00 〜 返却期限日当日 09:00（JST）」とする
- リマインド通知時刻のデフォルト値は `当日 07:00` とする
- 期限超過通知は期限日の翌日から返却完了まで毎日送信する
- 期限超過通知は土日祝を含めて送信する
- 期限超過通知の送信時刻はユーザー設定時刻を用いる

### 5.5 コメント要件
- 認証済みユーザーは書籍に対してコメントを投稿できる
- コメントは貸借機能の補助情報として扱う

## 6. 業務ルール
- 貸出可否は「認証状態」「Organization 所属」「`isActive`」「`discordId` 登録」「未返却/期限超過の有無」で判定する
- 管理者による個別貸出可否の上書きは MVP 対象外とする

## 7. MVP スコープ

### 7.1 MVP に含む
- GitHub OAuth 認証
- GeeKEN Organization メンバー判定
- Discord 連携情報を含むユーザー管理
- 貸出/返却管理（1人1冊、期限7日）
- 返却リマインド通知
- 期限超過の毎日通知

### 7.2 MVP に含まない
- 貸出延長機能
- 返却後ペナルティ
- 管理者による個別貸出可否の上書き

## 8. 非機能要件（最低限）
- 主要な状態変更（貸出、返却、期限超過判定、通知送信）は追跡可能であること
- 通知失敗時に再送可能な運用を考慮すること
